### 5. why data is not inserting into tables from data.sql file in resources
In newer versions of Spring Boot (2.5 and later), script-based initialization (like your data.sql file) is disabled by default.
To fix this and tell Spring Boot to execute the data.sql script, you need to add the spring.sql.init.mode=always property.
Additionally, because you are using Hibernate to create or update your database schema (spring.jpa.hibernate.ddl-auto=update), Spring needs to be told to run the data.sql file after Hibernate has finished creating the tables. You correctly had spring.jpa.defer-datasource-initialization=true for this!
 ```properties
 spring.sql.init.mode=always
 ```
